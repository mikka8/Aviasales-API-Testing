# ✈️ Aviasales API Testing Project

Проект посвящен самостоятельному независимому тестированию публичных API-методов сервиса Aviasales (Travelpayouts). В рамках проекта были спроектированы тест-кейсы, разработана небольшая коллекция базовых автоматизированных запросов в Postman, проведено тестирование бизнес-логики и граничных значений, а также зафиксированы обнаруженные дефекты.

## 🛠️ Используемые технологии и инструменты
* **HTTP-клиент:** Postman (для составления и выполнения запросов)
* **Язык автотестов:** JavaScript / Postman Sandbox
* **Документация и баг-трекинг:** Markdown, GitHub

## Реализация проекта
В ходе работы был проведен детальный анализ официальной спецификации методов /v3/get_latest_prices и /v1/prices/calendar. 
Документация методов доступна по адресу: https://clck.ru/3Trmrn. 

На основе требований, с применением техник эквивалентного разделения и анализа граничных значений, спроектирован полноценный тестовый набор для валидации параметров длин строк (коды IATA/ICAO), а также позитивных и негативных сценариев для фильтров дат и длительности поездки. 

Все проверки были успешно автоматизированы в среде Postman: создана структурированная коллекция запросов, написаны JS-скрипты для некоторых запросов (верификации статус-кодов, структуры JSON-ответов и времени отклика сервера). По результатам тестирования все выявленные отклонения от бизнес-логики были задокументированы в виде подробных баг-репортов с классификацией уровня серьёзности.

## Как запустить проект локально

**Предусловия**
1. Установленное десктопное приложение Postman или расширение для браузера.
2. Активный токен доступа (API-token), полученный в личном кабинете Travelpayouts (https://app.travelpayouts.com/profile/api-token).

**Шаги для запуска тестов:**
1. Клонируйте репозиторий или скачайте файл коллекции Aviasales_api_collection.json 

2. Откройте Postman и нажмите кнопку Import, выберите скачанный JSON-файл коллекции.

3. Перейдите в настройки коллекции, нажав на папку Aviasales (вкладка Variables).

4. Вставьте Ваш персональный токен в поле "Value" для переменной api-token.

5. Все готово для запуска тестов.
