# Баг-репорты, найденные по тестированию методов

## BUG-001
| Поле | Значение |
|:-----|:---------|
| **ID** | BUG-001 |
| **Краткое описание** | Игнорирование фильтрации по длительности поездки (параметр trip_duration) |
| **Модуль** | /v1/prices/calendar |
| **Приоритет** | High |
| **Серьёзность** | Major |
| **Окружение** | Postman Client, API: http://api.travelpayouts.com |
| **Предусловия** | 1. Пользователь имеет аккаунт в Travelpayouts (travelpayouts.com).<br>2. Имеется API-ключ для доступа к методам (app.travelpayouts.com/profile/api-token).<br>3. Установлен клиент с поддержкой импорта коллекций Postman/OpenAPI.<br>4. API-ключ добавлен в переменную "api-token" в родительской папке коллекции (Aviasales). |
| **Шаги воспроизведения** | 1. Открыть метод /v1/prices/calendar.<br>2. Передать следующие параметры: origin=LED, destination=AER и departure_date=2026-07.<br>3. Передать параметр фильтрации trip_duration=1.<br>4. Выполнить GET-запрос. |
| **Ожидаемый результат** | Запрос возвращает статус 200 OK. В массиве данных data находятся только те авиабилеты, у которых длительность поездки (разница между датой вылета и возвращения) строго равна 1 дню. |
| **Фактический результат** | Запрос возвращает статус 200 OK. Фильтрация не применяется: в теле ответа возвращается стандартный массив объектов с длительностью поездок, не соответствующей установленному параметру trip_duration (фильтр полностью проигнорирован): разница в днях между параметрами "departure_at" и "return_at" более одного дня. |
| **Статус** | Open |

## BUG-002
| Поле | Значение |
|:-----|:---------|
| **ID** | BUG-002 |
| **Краткое описание** | Несоответствие фактического поведения API документации при отправке GET-запроса без параметров origin и destination |
| **Модуль** | /aviasales/v3/get_latest_prices |
| **Приоритет** | Medium |
| **Серьёзность** | Normal |
| **Окружение** | Postman Client, API: http://api.travelpayouts.com |
| **Предусловия** | 1. Пользователь имеет аккаунт в Travelpayouts (travelpayouts.com).<br>2. Имеется API-ключ для доступа к методам (app.travelpayouts.com/profile/api-token).<br>3. Установлен клиент с поддержкой импорта коллекций Postman/OpenAPI.<br>4. API-ключ добавлен в переменную "api-token" в родительской папке коллекции (Aviasales). |
| **Шаги воспроизведения** | 1. В папке "/latest_prices" коллекции Postman найти запрос "Empty request".<br>2. Не указывать в параметрах значения origin и destination (отправить пустой запрос). <br>3. Выполнить GET-запрос. |
| **Ожидаемый результат** | Согласно документации: "Если origin и destination не указаны, по умолчанию будет возвращаться origin=MOW" запрос должен успешно обрабатываться со статусом 200 OK и возвращать результаты с вылетом из Москвы. |
| **Фактический результат** | Сервер возвращает ошибку валидации со статусом 400 Bad Request. Запрос отклоняется. |
| **Статус** | Open |
